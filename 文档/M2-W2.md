**login 等价类覆盖（W2 用例设计核心）**：

| 等价类 | 测试方法 | 预期 |
|--------|----------|------|
| 用户名存在 + 密码正确 + 状态启用 | `login_success_returnsEmployee` | 返回 `Employee` |

### 3. 测试方法（W2）

| 层级 | 工具/框架 | 做法 |
|------|-----------|------|
| Controller | JUnit 5 + MockMvc + `@WebMvcTest` | Mock `EmployeeService`，断言 HTTP 状态与 `$.code` / `$.msg` / `$.data` |
| Service | JUnit 5 + Mockito `@ExtendWith` | `@InjectMocks` + `@Mock Mapper`，`assertThrows` 验证异常分支 |
| Mapper | `@MybatisTest` + H2 `MODE=MySQL` | 真实执行 XML SQL，断言行数与约束异常 |
| 用例设计 | **等价类划分**（login） | 四类业务结果各至少 1 条用例；密码错误用例体现**边界值**思想（与正确值相邻） |

### 4. 遇到的问题与解决方案

| 问题 | 现象 | 解决方案 |
|------|------|----------|
| 测试目录被 git 忽略 | `src/test` 无法 push | 修改根目录 `.gitignore`，保留 `src/test` |
| Mapper 测 H2 无表 | 启动报 table not found | 新增 `schema-test.sql`，`application-test.yml` 配置 `spring.sql.init.mode=always` |
| Controller 登录测 Token | `@WebMvcTest` 未加载完整 JWT 配置 | `@MockBean JwtProperties`，`@BeforeEach` 注入 secret/ttl |
| H2 与 MySQL 语法差异 | 个别类型/函数不一致 | H2 URL 加 `MODE=MySQL`；表结构按业务表精简字段 |
| 合并 dev 后依赖缺失 | 他人分支未带 test 依赖 | 在 `sky-server/pom.xml` 显式声明测试依赖（与 M1 协调后的 `fix-一些依赖缺失` 一并解决） |
