## 1.2 系统概述

苍穹外卖（Sky Take-Out）是一款基于 Spring Boot 3.1.2 + MyBatis + MySQL + Redis 技术栈开发的在线外卖订购系统。系统面向两类用户角色：管理员（商家端）和普通用户（C端），提供完整的外卖业务闭环。

**管理端功能**包括：员工管理（登录/退出/增删改查/状态管理）、分类管理、菜品管理（含口味配置）、套餐管理、订单管理（接单/拒单/派送/完成）、报表统计（营业额/用户统计/订单统计）及工作台数据概览。

**C端功能**包括：微信用户登录、地址簿管理、购物车管理、下单支付（微信支付集成）及历史订单查询。

**技术架构**采用典型的三模块分层设计：

- **sky-common**: 公共组件模块，包含通用工具类、常量定义、异常处理、拦截器等。
- **sky-pojo**: 实体类模块，定义数据实体、DTO、VO等。
- **sky-server**: 业务主模块，包含Controller层、Service层、Mapper层及配置，是系统的核心。

系统采用前后端分离架构，后端通过 RESTful API 与前端（管理端Web + C端微信小程序）进行交互。数据层使用 MySQL 8.0 作为持久化存储，Redis 7 用于缓存与会话管理，文件存储使用阿里云 OSS 对象存储。

## 3.2 性能测试概述

### 3.2.1 测试策略

性能测试由 M8 负责执行，使用 Apache JMeter 5.x 对系统中5个高频核心接口进行多级别并发压力测试。测试目标为：

1. 评估系统在不同并发负载下的吞吐量（TPS）和响应时间表现。
2. 识别系统性能瓶颈，为后续性能优化提供数据支撑。
3. 验证系统在极限并发下的稳定性与错误率。

### 3.2.2 测试设计

| 序号 | 被测接口 | 接口说明 | 并发级别 | 测试指标 |
|---|---|---|---|---|
| 1 | `POST /admin/employee/login` | 管理端员工登录 | 50 / 100 / 200 / 500 | TPS、平均响应时间、P99、错误率 |
| 2 | `GET /admin/order/conditionSearch` | 管理端订单条件搜索 | 50 / 100 / 200 / 500 | TPS、平均响应时间、P99、错误率 |
| 3 | `POST /user/order/submit` | C端用户下单 | 50 / 100 / 200 / 500 | TPS、平均响应时间、P99、错误率 |
| 4 | `GET /user/dish/list` | C端菜品列表查询 | 50 / 100 / 200 / 500 | TPS、平均响应时间、P99、错误率 |
| 5 | `POST /admin/common/upload` | 文件上传（OSS） | 50 / 100 / 200 / 500 | TPS、平均响应时间、P99、错误率 |

### 3.2.3 测试方案

- **工具配置**：JMeter 5.x，线程组递增策略（Ramp-up 10秒，持续时间 120 秒）。
- **数据准备**：预置 1000 个测试员工账号、5000 条菜品数据、2000 条订单数据。
- **监控指标**：TPS、平均响应时间、P50/P90/P95/P99 响应时间、错误率、CPU/内存使用率。
- **压测流程**：预热（50并发）→ 常规负载（100并发）→ 中等压力（200并发）→ 极限压力（500并发），每级别独立执行。
