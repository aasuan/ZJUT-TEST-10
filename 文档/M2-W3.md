### 3. 测试方法对比（login）✅

对照《测试时间线》W3：M2 对 `POST /admin/employee/login` 做 **等价类划分法 vs 边界值分析法**。

| 方法 | 核心做法 | 在本模块落地 |
|------|----------|--------------|
| **等价类** | 按业务结果划 4 类：成功 / 账号不存在 / 密码错误 / 账号锁定 | W2：`EmployeeServiceImplTest` + `EmployeeControllerTest` 每类至少 1 条 |
| **边界值** | 在边界及邻近点取值：不存在用户、密码相邻错误值、status 0/1 | W2：`ghost`/`wrong-password`；W3 集成复验密码错误 + 成功路径 JWT |
| **对比结论** | 等价类保证分支不遗漏；边界值利于发现「差一点就对」的校验问题 | login 分支清晰，**以等价类为主框架**；边界值体现在错误密码与禁用状态；集成层只复验 EC-1 + EC-3，避免与 W2 重复堆砌 |

完整对照表见 [`w3-m2-集成测试总结.md`](./w3-m2-集成测试总结.md) 第 4 节。

### 4. W2 单元测试 vs W3 集成测试

| 对比项 | W2 单元测试 | W3 集成测试 |
|--------|------------|------------|
| 数据库 | H2 / Mock | Docker MySQL |
| Service/Mapper | Mock | 真实调用 |
| JWT 拦截器 | 未验证 | 验证 401 |
| 启动 | `@WebMvcTest` / `@MybatisTest` | `@SpringBootTest` 全上下文 |

### 5. 遇到的问题与解决方案

| 问题 | 现象 | 解决方案 |
|------|------|----------|
| MySQL 上误执行 H2 脚本 | 集成启动报 SQL/表冲突 | `application-integration.yml` 设置 `spring.sql.init.mode=never`；数据用 `@Sql(data-integration.sql)` 按方法注入 |
| MyBatis 重复映射 | 连续跑测报 Duplicate mapper | 先执行 `mvn clean test` |
| Docker 未就绪 | 连接 MySQL 失败 | `docker compose up -d`，等待 MySQL **healthy** 后再跑集成测 |
| Redis 依赖 | 全上下文启动需 Redis | 使用 M1 提供的 Compose（6380），与 `application-integration.yml` 一致 |
| 集成测数据污染 | 多次运行用户名冲突 | 新增员工用 `integ_m2_` + 时间戳保证唯一 |
