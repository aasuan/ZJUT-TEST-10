## 第2周：单元测试

### M2~M8（全员）

按 Controller → Service → Mapper 三层顺序编写：

| 层级 | 工具 | 目标 |
|---|---|---|
| Controller | MockMvc + Mock Service | HTTP 状态码、响应体 JSON 结构 |
| Service | Mockito Mock Mapper | 业务逻辑分支覆盖、异常抛出验证 |
| Mapper | @MybatisTest + H2 | SQL 正确性、唯一约束、分页查询 |

- [ ] 每写完一层跑一次 `mvn test`，确认通过
- [ ] 每完成一个 Task，Jira 拖入 Done
- [ ] 发现 Bug 立即建 Bug Issue，关联对应 Task

### M1（组长）

- [ ] 每日查看 Jira 进度，每天 Review 至少 1 人的测试代码
- [ ] 解决环境问题（如 H2 与 MySQL 语法差异、Mock 配置报错）
- [ ] 搭建 GitHub Actions CI：每次 Push 自动跑全量单元测试

### 第2周产出

> 单元测试全部通过，覆盖率 > 70%，CI 流水线跑通

## 第3周：集成测试 + 方法对比

### M2~M7

- [ ] **集成测试**：使用 `@SpringBootTest`，连接 Docker MySQL/Redis，验证全链路
- [ ] **测试方法对比**：选取 2 个核心接口，用两种不同方法重新设计用例并对比分析

| 成员 | 推荐对比方法 | 测试接口 |
|---|---|---|
| M2 | 等价类 vs 边界值 | login |
| M3 | 场景法 vs 因果图 | 菜品-分类关联 |
| M4 | 判定表 vs 等价类 | 套餐状态组合 |
| M5 | 状态迁移 vs 场景法 | 订单状态流转 |
| M6 | 等价类 vs 边界值 | 地址字段校验 |
| M7 | 场景法 vs 正交实验 | 下单全流程 |

### M1
